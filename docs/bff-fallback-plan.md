# BFF 전환 폴백/롤백 계획

이 문서는 BFF 전환 중 문제가 발생했을 때의 복구 기준과 절차를 정의합니다.

## 기본 원칙
- 기존 경로는 안정화 전까지 유지합니다.
- 전환은 기능 단위로 진행하며, 실패 시 해당 단위만 롤백합니다.
- 롤백은 최소 변경으로 빠르게 복구 가능한 방식으로 설계합니다.

## 롤백 트리거(예시)
- 인증/세션 실패율 증가(로그인/재발급 실패)
- 주요 쓰기 기능 실패(게시글/댓글/좋아요 등록 불가)
- 개인화 데이터 누락(마이페이지/프로필 조회 불가)
- 오류율 급증(5xx/4xx 비율 급증)

## 롤백 방식
- 최근 커밋 되돌리기(기능 단위 커밋 기준)
- 신규 Server Action 전환 비활성화 및 기존 경로 복원
- 관련 클라이언트 호출을 기존 경로로 즉시 회귀

## 폴백 전략
- 신규 Server Action 전환 전, 기존 호출 경로 유지
- 전환 실패 시 서비스 레이어에서 기존 경로로 즉시 회귀
- 문제 범위가 특정 리소스일 경우 해당 리소스만 폴백
- P0 인증/세션은 Server Action 실패 시 기존 `/api` 경로로 임시 회귀한다.

## 책임/검증
- 롤백 후 회귀 테스트 시나리오 재실행
- 장애 원인 분석 후 재전환 여부 결정
